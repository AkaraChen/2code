## 为什么选择 2code？

每个开发者都熟悉这种感觉：你正沉浸在心流中，进展顺利，然后——_alt-tab_。终端在这里。IDE 在那里。AI 聊天在别处。Git 历史在另一个窗口。每一次切换都是微小的上下文税，最终累积成流失的动量。

**2code 建立在一个简单的理念上：你的编码环境不应该打断你的心流。**

这是一个终端、AI 代理和 Git 共存的桌面工作空间——专门为 **vibe coding** 这门新兴实践而设计：在与 AI 协作的同时保持心流状态的艺术。

---

## 设计哲学

### 两位编码者，一个工作空间

"2code" 这个名字反映了其核心愿景：**你 + AI，作为伙伴一起工作**。

在 2code 中，AI 代理不是被塞进侧边栏或模态框里。它们有自己的标签页，就在你的终端旁边。你的 AI 伙伴拥有持久的记忆、完整的对话历史，并始终保持与你的项目上下文的连接。

### 永不丢失上下文

- **持久化终端**在应用重启后依然保留完整的滚动历史
- **代理对话**记得你上次停在哪里
- **Git 状态**始终可见、始终最新

应用记得一切，所以你不必记得。

### 专注于你的赛道

同时处理多个功能？每个 **profile** 创建一个隔离的 git worktree，拥有自己的终端会话和代理对话。不再需要切换分支时丢失终端历史或 AI 上下文。

### 心流状态，精心设计

2code 的每一个设计决策都服务于一个目标：**最小化上下文切换**。

- 实时 diff 查看器随时可访问
- 活动热力图让持续编码变得有趣
- 长时间运行的命令完成时音频通知
- 全程键盘优先导航

## 你可以做什么

**项目与 Profile**

- 将任何代码文件夹导入为项目
- 为每个功能或实验创建隔离的 worktree
- 每个 profile 支持自动设置/清理脚本

**终端**

- 每个 profile 支持多个标签页
- 可自定义主题、字体和大小
- 会话在重启后持久保存
- 命令完成时音频提醒

**AI 代理**

- 内置支持 Claude Code、Codex、Cursor 等
- 代理市场发现新的 AI 工具
- 丰富的聊天 UI，可视化工具调用
- 在同一对话中切换模型

**Git 集成**

- 实时 diff 查看器，显示暂存和未暂存的更改
- 提交历史浏览器，支持语法高亮
- 一键创建 Pull Request
- GitHub Desktop 集成

- 每天与 AI 编码助手一起工作
- 生活在终端中，但需要的不仅仅是一个 shell
- 同时处理多个功能分支

## 愿景

我们相信软件开发的未来是 **人 + AI 协作**——不是 AI 取代开发者，而是 AI 作为一个持久的伙伴，理解你的代码库，记住你的对话，在同一空间与你并肩工作。

2code 是我们构建那个空间的尝试。
