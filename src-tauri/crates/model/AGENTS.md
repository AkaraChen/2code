# AGENTS.md — src-tauri/crates/model

## OVERVIEW
All data types: Diesel ORM models, DTOs, and domain types. No logic — pure data definitions.

## FILES
| File | Role |
|------|------|
| `schema.rs` | Auto-generated Diesel schema — DO NOT EDIT manually |
| `project.rs` | `Project` (Queryable), `NewProject` (Insertable), `UpdateProject` (AsChangeset) |
| `profile.rs` | `Profile`, `NewProfile`, `UpdateProfile` |
| `pty.rs` | `PtySessionRecord` (Queryable), `NewPtySession`, `PtyOutputChunk` |
| `notification.rs` | `NotificationEntry`, `NotificationState` (shared with sidecar) |
| `watcher.rs` | `WatchEvent` DTO |
| `debug.rs` | `LogEntry` type |
| `error.rs` | `AppError` — unified error type returned from all Tauri commands |
| `lib.rs` | Re-exports |

## KEY NOTES
- `schema.rs` is generated by Diesel — run `diesel print-schema` or add migrations to regenerate
- `AppError` implements `serde::Serialize` so Tauri can send it to frontend as JSON
- `notification.rs` types are also used by the `2code-helper` sidecar (shared crate)
