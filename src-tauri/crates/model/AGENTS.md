# AGENTS.md — src-tauri/crates/model

## OVERVIEW
All data types: Diesel ORM models, DTOs, and domain types. No logic — pure data definitions.

## FILES
| File | Role |
|------|------|
| `schema.rs` | Auto-generated Diesel schema — DO NOT EDIT manually |
| `project.rs` | `Project` (Queryable), `NewProject` (Insertable), `UpdateProject` (AsChangeset). Plus all git DTOs: `GitChangeKind`, `IndexEntry`, `IndexStatus`, `GitCommit`, `GitAuthor`, `GitDiffStats`, `FileDiffSides`, `GitBinaryPreview`, `LogFilter`, `CommitRef`, `GraphEdge`, `GraphRow`, `BranchInfo`, `RemoteInfo`, `RemoteBranchInfo`, `TagInfo`, `StashEntry`, `ProjectConfig`, `ProjectWithProfiles` |
| `profile.rs` | `Profile`, `NewProfile`, `UpdateProfile` |
| `pty.rs` | `PtySessionRecord` (Queryable), `NewPtySession`, `PtyOutputChunk` |
| `notification.rs` | `NotificationEntry`, `NotificationState` (shared with sidecar) |
| `watcher.rs` | `WatchEvent` DTO |
| `debug.rs` | `LogEntry` type |
| `filesystem.rs` | `FileSearchResult` and other file-system DTOs |
| `topbar.rs` | Top-bar app DTOs |
| `git_error.rs` | `GitError` enum + `classify_stderr` heuristic mapper. Used by newer git ops that need structured error variants (auth failure vs. non-fast-forward etc.). `AppError::GitError` still carries a `String` for back-compat |
| `rewrite.rs` | History-rewrite DTOs: `RewritePlan`, `CommitAction` (Pick / Reword / Squash / Drop), `Identity`, `RewriteOutcome`. Drives the unified `git rebase -i` engine in `infra::git::rewrite` |
| `error.rs` | `AppError` — unified error type returned from all Tauri commands |
| `lib.rs` | Re-exports |

## KEY NOTES
- `schema.rs` is generated by Diesel — run `diesel print-schema` or add migrations to regenerate
- `AppError` implements `serde::Serialize` so Tauri can send it to frontend as JSON
- `notification.rs` types are also used by the `2code-helper` sidecar (shared crate)
- All git DTOs in `project.rs` use `Serialize + Deserialize + Clone + Debug + PartialEq + Eq` so they can flow over the Tauri IPC boundary and be diffed in TanStack Query caches
- When adding a new git DTO, also extend the matching TS interface in `src/features/git/changesTabBindings.ts` (or regenerate via `cargo tauri-typegen generate` once that's the source of truth)
